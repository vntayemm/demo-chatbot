    TEAM PROFILE - DEMO-CHATBOT DELIVERY TEAM¶

    1. Muc dich¶

    Tai lieu nay mo ta cau truc team, vai tro, va co che phoi hop de trien khai du an chatbot theo huong production-ready.

    2. Co cau team de xuat¶

    2.1 Core team¶

    • Project Manager (PM)
    • Product Owner / Business Analyst (PO/BA)
    • Tech Lead
    • Backend/AI Engineer
    • Frontend Engineer
    • QA/UAT Coordinator
    • DevOps/SRE Engineer

    2.2 Extended team (neu can)¶

    • Security/Compliance Specialist
    • Solution Architect
    • Customer Success / Training Specialist

    3. Mo ta vai tro chinh¶

    Project Manager (PM)¶

    • Dieu phoi tong tien do, pham vi, rui ro
    • To chuc weekly governance va escalation
    • Dam bao deliverables dung han

    Product Owner / BA¶

    • Chot yeu cau nghiep vu va acceptance criteria
    • Dieu phoi UAT va sign-off voi business
    • Quan tri backlog nghiep vu

    Tech Lead¶

    • Chiu trach nhiem kien truc va chat luong ky thuat
    • Dinh huong thiet ke API/retrieval/performance
    • Ho tro quyet dinh ky thuat critical

    Backend/AI Engineer¶

    • Xay dung endpoint chat theo domain
    • Toi uu retrieval va quality response
    • Tich hop logging/metrics trong backend

    Frontend Engineer¶

    • Xay dung va cap nhat giao dien chat/landing
    • Dam bao flow UI -> API on dinh
    • Ho tro test va handover frontend

    QA/UAT Coordinator¶

    • Lap test plan va test report
    • Dieu phoi UAT cases, tong hop defect
    • Theo doi trang thai pass/fail theo tieu chi

    DevOps/SRE Engineer¶

    • Trien khai moi truong staging/production
    • Cau hinh monitoring/alerting
    • Ho tro go-live va incident handling

    4. Team capacity tham chieu¶

    Vai tro So luong de xuat Muc do tham gia
    PM 1 Full/Part-time theo goi
    PO/BA 1 Part-time (cao diem UAT)
    Tech Lead 1 Part-time
    Backend/AI Engineer 1-2 Full-time
    Frontend Engineer 1 Part-time
    QA/UAT 1 Part-time, cao diem truoc go-live
    DevOps/SRE 1 Part-time, cao diem go-live

    5. Co che phoi hop¶

    • Daily sync (neu can): 15 phut
    • Weekly project sync: 30-60 phut
    • Steering review: 2-4 tuan/lan
    • Incident bridge: kich hoat khi P1/P2

    6. RACI tom tat¶

    Workstream PM PO/BA Tech Lead Dev Team QA DevOps
    Scope/KPI A R C I I I
    Build C I A R C C
    Test/UAT C A C R R I
    Go-live R A C C C R
    Operations C C C C I A/R

    7. Nguong escalation¶

    • Blocker > 24h: PM escalate Tech Lead + PO
    • Rui ro anh huong milestone: escalate Steering
    • Sev1 incident: kich hoat incident runbook ngay

    8. Dau ra theo role (deliverables)¶

    • PM: status report, risk log, action tracker
    • PO/BA: requirement baseline, UAT sign-off
    • Tech Lead: architecture decisions, technical review
    • Dev team: source code, API docs, release notes
    • QA: test report, defect log, UAT summary
    • DevOps: deployment record, monitoring dashboard
