Đã cài và chạy MLflow self-hosting thành công trên máy bạn.

- MLflow được cài trong virtualenv tại: `/Users/admin/Documents/GitHub/AI-Installation/.venv`
- Server đang chạy tại: [http://127.0.0.1:5001](http://127.0.0.1:5001)
- Trạng thái kiểm tra HTTP: `200 OK`
- Backend mặc định đang dùng: `sqlite:///mlflow.db` (đúng theo tài liệu mới của MLflow self-hosting)

Lý do dùng `5001` thay vì `5000`: cổng `5000` trên máy bạn đã bị dịch vụ khác chiếm, nên mình chuyển cổng để chạy ổn định ngay.

Nếu bạn muốn chạy lại thủ công:
```bash
cd /Users/admin/Documents/GitHub/AI-Installation
./.venv/bin/mlflow server --host 127.0.0.1 --port 5001
```

Nếu muốn dừng server:
```bash
pkill -f "mlflow server --host 127.0.0.1 --port 5001"
```

Tham chiếu tài liệu self-hosting đã áp dụng: [MLflow Self Hosting Overview](https://mlflow.org/docs/latest/self-hosting/)

Đã cài xong theo lệnh bạn yêu cầu trong virtualenv của project.

- `mlflow`: đã có sẵn, version `3.11.1`
- `optuna`: đã cài mới, version `4.8.0`

Lệnh đã chạy:
```bash
./.venv/bin/python -m pip install mlflow optuna
```

Nếu bạn muốn dùng ngay trong terminal:
```bash
source .venv/bin/activate
python -c "import mlflow, optuna; print(mlflow.__version__, optuna.__version__)"
```

Lần này `mlflow` vẫn OK, nhưng `torch`/`torchvision` chưa cài được vì môi trường hiện tại đang dùng Python `3.13`:

- `mlflow`: đã có (`3.11.1`)
- `torch`: `No matching distribution found for torch`
- Nguyên nhân chính: PyTorch hiện chưa phát hành wheel tương thích cho Python `3.13` trên môi trường này.

Cách chạy được ngay (khuyến nghị): tạo venv Python `3.11` rồi cài lại:

```bash
python3.11 -m venv .venv311
source .venv311/bin/activate
python -m pip install --upgrade pip
python -m pip install mlflow torch torchvision
```

Nếu bạn muốn, mình làm luôn bước tạo `.venv311` và cài full cho bạn ngay.
